# 자바

### 자바 특징
 - 운영체제에 독립적이다.
 - 객체지향언어이다
 - 자동 메모리 관리(포인터 ,메모리 관리) 를 한다.
 - 동적 로딩지원을 한다.
 - 멀티쓰레드 지원을 한다.
 - 네트워크와 분산환경 지원을 한다.
 - 자바는 컴파일언어여서 타입을 명확하게 명시해야한다.

### 자바 언어 장점
 - 코드 작성이 쉽다
 - 객체지향언어로서 기존 코드 재사용 가능하다
 - 보안에 강하다
 - 플랫폼에 독립적이다.

### 자바 단점
 - 실행 속도가 C와 C++ 보다 느리다
 - 백업 작업에 적합하지 않는다. (자바는 주로 저장에 중점)
 - 이해하기 힘든 긴 코드가 많다.



## 변수
 > 수학)  변할수 있는 값
 - 프로그래밍에서는 컴퓨터메모리(RAM)에 값을 기록하기 위한 공간이다.
 
### 변수선언
  - 메모리공간에 값을 할당받는 것
### 변수접근
  - 메모리에서 값을 가져오는 것 
     - 사용시 유지 보수성 증가

### 변수 규칙
 - 대소문자 구분
 - 숫자로 시작할수 없다
 - $과_이외의 특수문자는 사용할수없다
 - 키워드는 변수의 이름으로 사용할 수 없다.

### 변수 자료형 결정
 - 해당 변수에 값을 저장 및 참조하는 방식의 결정

## 상수
  > 수학) 변하지 않는 값
  - 프로그래밍에서는 한번만 값을 할당 할 수 있는 메모리
  - 상수의 이름은 모두 대문자로 짓는 것이 관례

### 리터럴
 - 변수에 대입되는 **값** 자체


## 형변환
 - 작은타입->큰타입 :자동
 - 큰타입 ->작은타입 :강제 (데이터 손실)


### 객체는 메모리 공간이 유동적으로 정해진다.
### 객체변수는 주소공간을 가리킨다.
